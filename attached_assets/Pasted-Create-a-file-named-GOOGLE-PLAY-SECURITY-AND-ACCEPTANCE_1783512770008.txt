Make Ciyato safer, more transparent, and more likely to satisfy Google Play review expectations.

1. What Ciyato is

- app library organization
- smart categories
- file/photo/search companion features
- optional personalization
- user-controlled experience

2. Core trust principle
- privacy first
- minimum permissions
- transparent behavior
- no misleading claims
- no silent uploads
- no deceptive device-control language
- user can switch back
- user can disable features
- no harmful behavior

3. Permission strategy
For every permission, document:
- why it is needed
- when it is requested
- whether it is optional

- what happens if denied

Include examples like:
- launcher/home role
- installed app visibility / query permissions
- photo access
- file access
- notification access if ever used
- location only if truly required for weather
- accessibility permissions only if ever needed and only if policy-compliant

4. Permissions we must avoid unless absolutely necessary
Explain risky permissions and why they should not be used casually.

5. Google Play compliance mindset
Document principles such as:
- least privilege
- user benefit must be clear
- permission use must match core functionality
- permission rationale must be contextual
- app must not appear deceptive
- privacy policy must match actual behavior
- data safety declarations must be accurate
- onboarding must not overclaim
- no vague “AI needs everything” logic

6. Security architecture expectations
Document:
- no hardcoded secrets
- secure local storage where needed
- encrypted sensitive preferences where needed
- account/session safety if login is added
- no exporting private user data without consent
- safe logging practices
- debug logs removed/reduced in release builds
- crash handling without leaking sensitive information

7. Privacy architecture expectations
Document:
- on-device-first approach
- file/photo scanning performed locally where possible
- if cloud AI is added later, it must be opt-in and transparent
- user data categories
- what is stored locally
- what is not collected
- deletion and reset controls

8. UX for trust
Document:
- onboarding must explain permissions clearly
- setup flow must reassure the user
- settings must show what features are active
- users must be able to switch back from Ciyato launcher
- users must be able to reset preferences
- users must understand what “hidden apps” and “removed from display” mean
- nothing should imply uninstalling when it only hides
